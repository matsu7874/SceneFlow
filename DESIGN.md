# SceneFlow デザインシステム

SceneFlow 全画面で共有する視覚言語の正規リファレンス。新規・改修時はこのルールに準拠する。
本書は各画面リデザインを担当するサブエージェントへの仕様書も兼ねる。

## 1. コンセプト

**「精密な計器（instrument）」**。イマーシブシアターの複雑な因果・動線・状態を扱うための
**情報密度が高く、可読性と操作性を最優先する**ツール。装飾は最小限、線とタイポと余白で秩序を作る。

- 明るい中立基調（クールグレー）＋ 1つの確信ある青アクセント
- データ（時刻・ID・座標・数値）は等幅で整列
- 1px の罫と一貫した余白スケールで「設計図」のような精密さ
- エンティティ色（人物・場所のメンバーカラー）は可視化のためのアクセントとして活かす

避けるもの: 過度な装飾、影の多用、丸すぎる角、彩度の高い背景、明朝系の演出的フォント。

## 2. デザイントークン（`src/index.css` の `:root` で定義）

既存の `--color-*` / `--shadow-*` は**値を新パレットに張り替え済み**。各画面はこれらを参照するだけで新ルックになる。
新規実装では下記の意味別トークンを使う。

### 色

| トークン                                         | 値        | 用途                             |
| ------------------------------------------------ | --------- | -------------------------------- |
| `--app-bg`                                       | `#eef1f5` | アプリシェル（ページ背後）の背景 |
| `--color-background`                             | `#ffffff` | カード・パネル・入力欄の面       |
| `--color-surface`                                | `#f6f7f9` | わずかに沈んだ面・補助ブロック   |
| `--color-surface-hover`                          | `#eef1f5` | ホバー面                         |
| `--color-surface-dark`                           | `#e4e8ee` | ヘッダー帯・インセット           |
| `--ink` / `--color-text-primary`                 | `#18202c` | 主要テキスト                     |
| `--ink-2` / `--color-text-secondary`             | `#566173` | 補助テキスト・ラベル             |
| `--ink-3` / `--color-text-disabled`              | `#8a93a3` | 弱いテキスト・プレースホルダ     |
| `--line` / `--color-border`                      | `#e3e7ed` | 罫線・境界                       |
| `--line-strong` / `--color-border-hover`         | `#cdd3dc` | 強い罫・区切りの主線             |
| `--accent` / `--color-primary`                   | `#2563eb` | 主アクセント・操作要素・選択     |
| `--accent-strong` / `--color-primary-dark`       | `#1d4ed8` | ホバー・押下                     |
| `--accent-soft`                                  | `#e8f0fe` | アクセントの淡い面（選択背景等） |
| `--ok` `#16a34a` / `--ok-soft` `#dcfce7`         |           | 成功・正常                       |
| `--warn` `#d97706` / `--warn-soft` `#fef3c7`     |           | 警告                             |
| `--danger` `#dc2626` / `--danger-soft` `#fee2e2` |           | エラー・破壊的操作               |

### タイポグラフィ

- `--font-sans`: `'IBM Plex Sans JP', ...` — UI 全般・見出し（技術的で精密な和文ゴシック）
- `--font-mono`: `'IBM Plex Mono', ...` — **時刻・ID・座標・数値・コード**は必ず等幅
- 見出しは sans の太さで階層を作る（明朝は使わない）
- スケール（目安）: ページ見出し 1.5rem/700、セクション 1.05rem/600、本文 0.9rem/400、
  ラベル・overline 0.7rem/600・letter-spacing 0.08em・大文字 or 弱色

### 余白・形状

- 余白: `--space-1..6` = 4 / 8 / 12 / 16 / 24 / 32px。基本リズムは 8px グリッド
- 角丸: `--r-sm` 6px / `--r` 8px / `--r-lg` 12px / `--r-pill` 999px
- 影: `--shadow-1`（面の浮き）/ `--shadow-2`（カード）/ `--shadow-pop`（ポップオーバー）。多用しない
- 罫線は基本 1px `--line`。区切りの主線は `--line-strong`

## 3. コンポーネント規約

- **ボタン**: 主＝`--accent` 背景/白文字、副＝白背景 `--line` 枠/`--ink` 文字、破壊的＝`--danger`。
  角丸 `--r`、padding `0.5rem 1rem`、`:focus-visible` は `--accent` の 3px リング。
- **入力/セレクト**: 白背景・`--line` 枠・角丸 `--r-sm`、focus で枠 `--accent`＋淡いリング。
- **カード/パネル**: 白背景・`--line` 枠・角丸 `--r-lg`・`--shadow-2` は控えめに。
- **テーブル/リスト**: ヘッダー帯 `--surface-dark`、行は 1px `--line` 区切り、ホバー `--surface-hover`、
  数値列は等幅・右 or 整列。情報密度を優先（行高は詰め気味）。
- **バッジ/チップ**: `--r-pill`、淡い面＋濃いめ文字。状態色は status トークンで。
- **空状態**: 中央寄せ・アイコン・1行見出し＋補助文、点線 `--line` 枠のカード。
- **セクション見出し**: overline（弱色・字間広め）＋見出し。必要なら左に 2px のアクセント縦罫。
- **フォーカス可視性**: 全インタラクティブ要素に `:focus-visible` を必ず付与（キーボード操作前提）。

## 4. レイアウト

- ページは `.page`（padding・スクロール）→ 必要なら `.page-content`（カード面）。グローバルは `App.css`。
- 計器的な「列見出し＋整列行」を多用。CSS Grid の列定義をヘッダーと行で共有して整列させる。
- レスポンシブ: 狭幅では副次列を畳み、主要情報（識別子・状態・名前）を残す。

## 5. 所有権（コンフリクト回避）

- **グローバル層（Phase 1 が所有・サブエージェントは編集禁止）**:
  `src/index.html` / `src/index.css` / `src/App.css` / `src/components/Navigation/Navigation.module.css`
- **各サブエージェントの担当**: 自分の画面の `*.module.css` と関連 `*.tsx`（ページ含む）**のみ**。
  グローバル CSS は触らない。レイアウト変更が必要なら自分の module.css にクラスを追加して対応する。
- 既存のクラス名・`aria-label`・テスト参照セレクタは維持する（E2E/単体テストが依存）。

## 6. 画面別メモ

- **QuickLog（実装済み・基準）**: 列整列タイムライン、人物スウォッチ＋場所色チップ、計器的入力バー。
- **Entities**: 左リスト＋右フォーム。リストは型バッジ＋名前＋ID（等幅）。フォームはセクション分け、入力規約準拠。
- **MapEditor / Spatial**: SVG/Canvas。背景 `--surface`、ノード面 `--color-background`、罫 `--line`、
  パス/接続は `--ink-3`、選択・ハイライトは `--accent`。凡例・ツールバーをカード化。
- **Causality**: ノード＋依存エッジ。種別で色分けする場合は status/accent トークンで統一。等幅で時刻・ID。
- **Relationships**: 関係ダイアグラム＋エディタ。人物色（`person.color`）をノードに反映。
- **Validation**: 診断リスト。重大度を status 色（error/warn/ok）でバッジ化、件数サマリ、フィルタ。
- **Simulation**: タイムライン再生・速度・イベントログ。再生コントロールを計器パネル化、時刻は等幅。

## 7. 検証

実装後 `npx tsc --noEmit` / 該当 `npx vitest run` / `npx eslint <変更ファイル>` を通し、
開発サーバ（`npm run dev`）＋スクショで可視性・操作性・トークン準拠を確認する。
