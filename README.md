# SceneFlow - イマーシブシアターシミュレーター

SceneFlowは、複数の登場人物が異なる場所で同時進行する物語を設計・シミュレーションするためのツールです。イマーシブシアター（没入型演劇）や複雑なストーリーの構造を可視化し、検証することができます。

## 🎭 SceneFlowとは

イマーシブシアターでは、観客が自由に移動しながら物語を体験します。複数の登場人物が同時に異なる場所で行動するため、物語の全体像を把握することが困難です。SceneFlowは、このような複雑な物語構造を：

- **書きながら設計** - 「いつ・誰が・どこで・何をした」を綴るだけで物語データになる
- **視覚的に組み立て** - 空間ワークスペースで場所の配置・接続と人物の動線を編集
- **リアルタイムシミュレーション** - 実際の上演のように物語を再生
- **検証とデバッグ** - 矛盾や破綻（動線・所持・情報・生死）を自動検出

1. ブラウザで https://matsu7874.github.io/scene-flow/ にアクセス
2. サンプル（桃太郎・学院ミステリ・嵐の洋館ミステリ）を読み込むか、イベント入力で書き始める

```bash
# リポジトリをクローン
git clone https://github.com/matsu7874/scene-flow.git
cd scene-flow

# 依存関係をインストール
pnpm install

# 開発サーバーを起動
pnpm run dev
```

ブラウザで http://localhost:3000 を開きます。

## 📖 使い方

ナビゲーションは制作の流れに沿って「① 書く → ② 組む → ③ 検証・分析」の3フェーズに分かれています。編集内容はブラウザ（localStorage）に自動保存され、`Ctrl+Z` / `Ctrl+Shift+Z` でどの画面からでも取り消し・やり直しができます。

### ① 書く

#### イベント入力（/log）

「いつ・誰が・どこで・何をした」を1行ずつ記録すると、そのまま物語データになります。データの事前読み込みは不要で、ここから書き始められます。移動Actなしの場所変化には動線バッジが表示されます。

#### データ入出力（/data）

物語データ（JSON）の貼り付け・読み込み・書き出しと、サンプルの投入を行います。読み込んだデータは全画面で共有されます。

### ② 組む

#### エンティティ編集（/entities）

人物・場所・道具・情報・行動を作成・編集します。

- **人物** - 名前、色、属性（年齢・職業・性格）、他の人物との関係性
- **場所** - 名前、接続、タイプ、収容人数、移動時間（travelTime）、施錠（isLocked / requiredItem）
- **道具** - 大道具・小道具の区分、持ち運び可能・消費可能・組み合わせ可能などの属性
- **情報** - 内容、カテゴリ（事実・噂・秘密など）
- **行動** - 実行者、場所、時刻、種別、説明

Act の説明文から形態素解析でエンティティ候補を抽出する機能もあります。

#### 空間（/space）

場所の配置・接続を編集しながら、人物の動線と破綻地点を同じ地図の上で確認できます。編集はそのまま物語データに反映され、自動保存されます。

- **場所を追加** - 空白をダブルクリック
- **移動** - ノードをドラッグ
- **接続** - 「接続」ボタン → 2つのノードを順にクリック（または Ctrl/⌘+クリック）
- **編集・削除** - ノードをダブルクリックで編集、選択して Delete で削除
- **表示** - ホイールでズーム、空白ドラッグでパン、ミニマップ・グリッド表示
- **自動レイアウト** - グリッド配置・円形配置・力学配置（自動整列）
- **重ねて確認** - 「動線」「破綻」トグルで人物の動きと破綻地点を地図に重ねる。破綻の詳細パネルから場所をクリックすると地図とハイライトが連動
- **背景画像** - 間取り図などを下敷きにできる

#### 関係性（/relationships）

登場人物の相関図を俯瞰します（閲覧専用）。関係の編集はエンティティ編集で行います。

### ③ 検証・分析

#### 検証（/validation)

物語データの整合性をチェックします。因果・動線・所持・情報を時系列に再生し、破綻を自動検出します。

検出される問題（カテゴリー）:

- **生死・意識（エラー）** — 死亡・昏倒した人物がその後に行動する、死亡者を蘇生する、死亡者に情報を伝える、既に死亡した人物を再び殺害する等（被害者の状態の整合）
- **位置・動線（エラー）** — 隣接していない場所への直接移動、移動なしでの別場所での行動
- **施錠・鍵（エラー）** — 施錠された場所へ、必要な鍵（`properties.requiredItem`）を所持せずに入る／鍵未設定の施錠空間に入る（密室トリックの検証）
- **アイテム所持（エラー）** — 未所持の道具の受け渡し・使用、現場に無い道具の取得、`isPortable:false`（据付・大道具）の持ち去り
- **移動時間・アリバイ（警告）** — 直前の行動から、移動先の所要時間（`travelTime`）に満たない移動（アリバイ崩しの検算）
- **同時刻・共在（警告）** — 同時刻に複数の場所へ存在（分身）、その場にいない相手への対人行動
- **証言の矛盾（警告）** — 同一の論点（`subject`×`aspect`）で食い違う情報を同一人物が同時に保持（嘘と真実の衝突）
- **情報の知識（情報）** — 知らないはずの情報を話す

#### シミュレーション（/simulation）

物語を実際の上演のように再生します。

- **再生/一時停止・速度調整・タイムライン** - 任意の時刻にジャンプ
- **場所別ビュー** - 各場所にいる人物をリアルタイム表示
- **実行ログ** - 発生した行動を時系列で表示

#### 容疑者・機会（/opportunity）

「誰がやれたか」を逆引きするビューです。筋書きの検算だけでなく、フェアな手がかり配置・消去法の設計に使えます。

- **場所×時刻** — 任意の時刻・場所を選ぶと、そこに居た人物（容疑者）と、他の場所に居た人物（アリバイ）を一覧。被害者の生死・意識状態も表示
- **凶器・道具** — 道具を選ぶと、全期間を通じて「触れ得た人物」（所持していた／その道具のある場所に居合わせた）を一覧
- **情報・秘密** — 情報を選ぶと、「知り得た人物」を初めて知った時刻つきで一覧（伝聞の連鎖を含む）

#### 因果関係ビュー（/causality）

事実・証言の依存グラフを表示します。ノードを選択すると上流・下流が強調されます。

### 行動の種別（`act.type`）

`MOVE` / `TAKE` / `GIVE` / `DROP` / `USE` / `LEARN` / `SPEAK` に加え、事件を表す状態変化アクトを利用できます。いずれも被害者を `interactedPersonId` で指定します。

- `ATTACK` — 攻撃（対象を負傷させる）
- `INCAPACITATE` — 昏倒させる（対象は行動不可・`WAKE` で回復可能）
- `KILL` — 殺害（対象は以後行動不可・不可逆）
- `WAKE` — 蘇生・覚醒（昏倒・負傷から回復。死亡者には無効）

## 📊 データ形式

### 基本構造

```json
{
  "persons": [...],      // 登場人物
  "locations": [...],    // 場所
  "props": [...],        // 小道具
  "informations": [...], // 情報
  "acts": [...],         // 行動
  "initialStates": [...] // 初期配置
}
```

### 詳細な例

```json
{
  "persons": [
    {
      "id": 1,
      "name": "主人公",
      "color": "#FF0000",
      "description": "物語の主人公",
      "age": 25,
      "occupation": "探偵",
      "personality": "冷静沈着",
      "relationships": [
        {
          "targetId": "2",
          "type": "協力者"
        }
      ]
    }
  ],
  "locations": [
    {
      "id": 1,
      "name": "事務所",
      "connections": [2],
      "description": "探偵事務所",
      "type": "indoor",
      "travelTime": 5,
      "properties": { "isLocked": true, "requiredItem": "101" },
      "x": 100,
      "y": 200
    }
  ],
  "acts": [
    {
      "id": 1,
      "personId": 1,
      "locationId": 1,
      "time": "09:00:00",
      "description": "事件の依頼を受ける"
    }
  ],
  "initialStates": [
    {
      "personId": 1,
      "locationId": 1,
      "time": "00:00:00"
    }
  ]
}
```

## 💡 活用例

### イマーシブシアター

- 複数の観客が同時に体験する物語の設計

## 🔧 トラブルシューティング

**Q: データがロードされない**

- JSONフォーマットが正しいか確認（データ入出力ページの検証エラーに行番号が表示されます）
- 必須フィールド（id, name等）が含まれているか確認

**Q: シミュレーションが動かない**

- actsに時刻（time）が設定されているか確認
- initialStatesが設定されているか確認
- 人物と場所のIDが正しく対応しているか確認

**Q: 空間ワークスペースで場所が重なって表示される**

- 「レイアウト ▼」から自動レイアウト（グリッド・円形・力学）を試す
