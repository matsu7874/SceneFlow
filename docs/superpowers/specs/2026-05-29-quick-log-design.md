# イベント入力ページ（Quick Log）設計

- 日付: 2026-05-29
- ステータス: 承認済み（実装計画待ち）

## 目的

「どこで・誰が・何をした」という記録を、頭の中にあるイベントの流れのままテンポよく入力できるようにする。
入力スピードを最優先とし、まず骨格（イベント列）を素早く作り、後から会話・受け渡し・情報共有・動線矛盾の修正といった詳細を肉付けする 2 段階のワークフローを実現する。

`Act`（`personId` = 誰が / `locationId` = どこで / `description`・`type` = 何をした / `time`・`startTime` = いつ）が記録単位であり、本機能はこの `Act` をテンポよく量産・編集する専用 UI を提供する。

## スコープ

### やること

- 新規ページ `/log`（ナビゲーション「イベント入力」）を追加
- 第1段階: クイック入力バーによるイベントの連射入力
- 第2段階: タイムライン + 詳細パネルによる肉付け編集
- 動線矛盾の軽量チェック（警告バッジ）

### やらないこと（YAGNI）

- `Act` 型・`StoryData` 型の変更（既存型をそのまま使う）
- 因果エンジン（`engine.ts`）の改修
- シミュレーション再生やマップ編集との新規連携
- 動線矛盾の自動修正（検出・警告のみ。修正は人間が詳細パネルで行う）
- 複数人同時編集・サーバ同期

## ワークフロー

### 第1段階：クイック入力バー（速度の核心）

画面上部に常駐する 1 行入力:

```
[ 誰が ▾ ] が [ どこで ▾ ] で [ 何をした________________ ] @[時刻] (追加)
```

速度のための仕掛け:

1. **誰が / どこで**: オートコンプリート付きコンボボックス（`EntityCombobox`）。
   - 直前に確定した値を **保持（sticky）**。連続入力で繰り返す人物・場所を再入力しなくてよい。
   - 候補一覧に一致が無い場合、その場で新規エンティティを作成（「+ 新規: 〇〇」を選択 / Enter）。
2. **何をした**: 自由テキスト（`description`）。最も高速な入力点。頭の中の言葉をそのまま打つ。
3. **時刻**: 直前に確定した Act の時刻 + 既定増分（既定 5 分）を自動補完。編集可・空欄可。
4. **確定操作**: 「何をした」入力中の Enter で確定。確定後はフォーカスが「何をした」に戻り、誰が/どこで/時刻は保持（時刻は増分後の値）される。これにより「打つ→Enter→打つ→Enter」の連射が可能。
5. 確定した Act は即座に下のタイムラインへ反映される。

### 第2段階：タイムライン + 詳細パネル

- クイック入力バーの下に、`startTime`（無ければ挿入順）でソートしたタイムラインを表示。
- 各行は `時刻 / 誰が / どこで / 何した` を表示。
- 行をクリックするとインライン展開し、`ActDetailPanel` で後から以下を追記・修正:
  - 会話内容（情報 / SpeakAct 系フィールド）
  - 受け渡したアイテム（`propId` + `interactedPersonId`）
  - 共有した情報（`informationId`）
  - タイプ分類（`type`）・時刻微調整（`startTime`/`time`）・場所修正（`locationId`）
- 行の削除が可能。

### 動線チェック

- ある人物が、その人物の直前の Act の場所と異なる場所に、移動を表す Act を挟まずに出現した場合、その行に警告バッジを表示する。
- 既存の `src/utils/validation.ts` のユーティリティを活用・拡張する。
- 検出のみ。修正は利用者が詳細パネルで `locationId` を直すか移動 Act を挿入することで行う。

### `useQuickLog` hook の責務

`AppContext` の `storyData` / `setStoryData` を通じて以下を提供する:

- `addAct(input)`: 新規 Act 追加。`id = max(既存acts.id) + 1`（空なら 1）で採番。
- `updateAct(id, patch)`: Act の部分更新。
- `deleteAct(id)`: Act 削除。
- `addPerson(name)` / `addLocation(name)`: その場新規作成。妥当な既定値（色・空 connections 等）を付与し ID 採番。作成した ID を返す。
- sticky 既定値（直前の personId / locationId / 時刻）の保持。
- 時刻自動増分（既定 +5 分。`startTime` を基準に計算し、`time` 文字列も同期生成）。
- 動線矛盾の検出（人物ごとに時刻順で場所遷移を走査し、移動 Act 無しの場所変化を抽出）。

すべての書き込みは `setStoryData({ ...storyData, acts/persons/locations: ... })` の不変更新で行う（既存パターン踏襲）。

### データモデル

既存の `Act` インターフェース（`src/types/StoryData.ts`）をそのまま使用し、型変更は行わない。

- ソートキー: `startTime`（分・number）。未設定時は配列内の挿入順を維持。
- `time`（文字列）と `startTime`（分）は入力時に相互に同期させる（時刻文字列 ↔ 分）。
- 新規 `Person`: `id` 採番・`name`・`color`（既定色を割当）。
- 新規 `Location`: `id` 採番・`name`・`connections: []`。

### ルーティング / ナビゲーション

- `src/App.tsx` にルート `/log` を追加（`<Route path="/log" element={<QuickLogPage />} />`）。
- `src/components/Navigation/Navigation.tsx` に「イベント入力」リンク（`to="/log"`）を追加。

## エラー処理・エッジケース

- `storyData` が `null`（データ未読込）の場合: 入力バーは無効化し、データ読込を促すメッセージを表示。
- 「何をした」が空のまま確定: 確定しない（バリデーションで無視）。
- 存在しない人物/場所名で確定: その場新規作成フローへ誘導。
- 時刻文字列のパース失敗: 自動増分はスキップし、利用者の手入力に委ねる。

## テスト方針（TDD）

- **単体テスト** `tests/components/useQuickLog.test.ts`（相当）:
  - `addAct` の ID 採番・sticky 既定値の引き継ぎ
  - 時刻自動増分（+5 分、`time`↔`startTime` 同期）
  - `addPerson`/`addLocation` の既定値・ID 採番
  - 動線矛盾検出ロジック（移動なしの場所変化を検出 / 移動ありは検出しない）
- **コンポーネントテスト** `QuickActInput`:
  - Enter 確定でフォーカスが「何をした」に戻り、誰が/どこでが保持される
  - 候補なし時の新規作成フロー
- **E2E（Playwright）**:
  - 連射入力（複数イベントを Enter 連打で追加）の操作感確認
  - 行展開 → 詳細追記の流れ

## 受け入れ基準

1. `/log` ページで、マウスを使わず「何をした」を打って Enter するだけで Act を連続追加でき、誰が/どこでが保持される。
2. 候補に無い人物/場所をその場で作成して即 Act に使える。
3. タイムライン行を開いて会話・アイテム・情報を後から追記できる。
4. 移動 Act を挟まない場所変化に警告バッジが出る。
5. 既存テスト・lint・型チェックが通る。
