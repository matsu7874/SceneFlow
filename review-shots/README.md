# イマーシブシアター作家視点レビュー — 検証アーティファクト

このディレクトリは、SceneFlow をイマーシブシアター作家の視点でレビューした際の
Playwright 実機検証スクリプトとスクリーンショットです。

## 検証環境メモ

Playwright 公式 CDN がネットワークポリシーで遮断されていたため、npm 経由で取得した
Chromium バイナリを `executablePath`（`_lib.mjs`）に渡して実機操作しています。
ローカルで再現する場合は `npx playwright install chromium` で取得したパスに置き換えてください。

実行前提: `npm run dev`（http://localhost:3000）が起動していること。

## スクリプト

- `_lib.mjs` — ブラウザ起動の共通ヘルパー
- `01-quicklog.mjs` — イベント入力からゼロ入力（自作の館ミステリ）
- `02-tour.mjs` — 入力データのまま全8タブを巡回
- `10-sample.mjs` — 学院ミステリ サンプルをロードし再生・検証・各ビュー
- `16-fixflow.mjs` — 位置破綻を「移動」種別に変えた際の警告メッセージ変化

## 主な所見（詳細は本文レビュー参照）

- 整合性検査エンジン（位置/共在/アイテム/情報＋証言矛盾）と因果関係ビューが強力
- 入力モデルと検査要求の乖離で初見は警告が出やすい（明示的な移動アクトが必要）
- 学院サンプルは座標未設定で空間ビューが空表示、relationships 未定義で相関図が線なし
- アイテム/情報検査は種別＝その他のため既定で眠っている
